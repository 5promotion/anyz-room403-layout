## 2026-04-21: 代表電話を新番号に一本化、旧番号は廃止

- **決定**: 代表電話は日興パレス201契約の新番号に一本化。千駄ヶ谷オフィスの旧番号は廃止。
- **根拠**:
  - 工事の関係で旧番号を御成門（ANYZ方面）に引けない
  - ソフトバンク光で新番号を既に利用中、NTTとの共存は設備的に困難（NTT確認済み）
  - 2番号運用のコスト・運用負荷を避ける
- **代替案**:
  1. ソフトバンクに2番号追加契約（設備的に可否未確認→確認中）
  2. 旧番号を新番号へ転送設定
  3. 旧番号を廃止し新番号に一本化 ← **採用**
- **影響**:
  - 旧番号が印刷物・Webサイトに残る場合、全面的な更新が必要
  - 既存取引先への周知が必要

## 2026-04-21: Fondesk（電話代行サービス）導入

- **決定**: 代表電話の一次受付にFondeskを契約。社員はテキスト（チャットツール）で用件を受け取る。
- **根拠**:
  - 代表電話を各社員が直接受ける運用は負荷が高い
  - 全社員へスマホ配布する前提で、チャット通知との相性が良い
  - 月額10,000円（50件まで）+ 超過110円/件で、規模に対して妥当
- **代替案**:
  - 有人受付専任配置 → 人件費が高い、却下
  - 留守電のみ → 顧客体験が悪い、却下
  - クラウドPBXのみ → 受付の一次対応が解決しない、Fondesk併用が最適
- **影響**:
  - 通知先チャットツールの選定が必要（未決定）
  - ソフトバンク光からFondesk指定番号への転送設定が必要

## 2026-04-21: 全社員への社用スマホ＋社用番号配布

- **決定**: 今後は全社員に社用スマホと社用番号を配布する方針
- **根拠**:
  - 私用端末・私用番号での業務利用リスクの排除
  - Fondeskからの通知を受けて折り返し発信する運用に必要
  - 業務連絡の一元化・退職時の情報分離が容易
- **代替案**:
  - BYOD（私用端末利用）+ 手当支給 → 情報管理リスク、却下
  - 一部社員のみ配布 → 運用ルールが複雑化、却下
- **影響**:
  - キャリア選定・月額コストの試算が必要（社員数・予算感の確認待ち）
  - クラウドPBX（代表番号を各スマホから発信可能にする）の併用検討が必要
  - MDM（端末管理）導入の検討が必要
