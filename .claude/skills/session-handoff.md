# セッションハンドオフ スキル

> セッション終了時にコンテキストを永続化する手順。
> ユーザーが「セッション終了」「引き継ぎ」「ハンドオフ」等を指示した際に実行する。

## 実行手順

### 1. セッションサマリーの作成

`memory/sessions/YYYY-MM-DD_概要.md` に以下を記録:

```markdown
# セッションサマリー: YYYY-MM-DD

## 実施内容
- （このセッションで行った作業を箇条書き）

## 重要な決定
- （意思決定とその根拠）

## 作成・変更したファイル
- （パスと変更内容）

## 未完了タスク
- （次セッションで継続すべき作業）

## 学んだこと・気づき
- （今後に活かせる知見）
```

### 2. active-context.md の更新

`memory/active-context.md` を最新状態に更新:

- **最終更新日時**を記入
- **現在の作業状態**を更新
- **直近の重要な決定**を反映
- **次セッションへの引継ぎ事項**を記載
- **未解決の課題**を更新

### 3. decisions.md の更新（該当時のみ）

重要な意思決定があった場合、`memory/decisions.md` に追記:

```markdown
## YYYY-MM-DD: 決定タイトル
- **決定**: 何を決めたか
- **根拠**: なぜその決定をしたか
- **代替案**: 検討した他の選択肢（却下理由含む）
- **影響**: この決定が及ぼす範囲
```

### 4. 知見の蓄積（該当時のみ）

案件横断で活用できる学びがあれば `learnings/insights.md` に追記。

### 5. HTML/可視化出力（オプション）

複雑な分析結果やアーキテクチャ図がある場合、htmlファイルとして `docs/` に保存。
ブラウザで閲覧可能な形式にすることで、Obsidian以外でも参照可能にする。

## 注意事項

- 機密情報（APIキー、パスワード等）はセッションサマリーに含めない
- ファイルが肥大化しないよう、サマリーは簡潔に（1ファイル100行以内目安）
- 3ヶ月以上前のセッションサマリーは定期的にアーカイブを検討
